```
|            |  ─────────────────────────────         |
|            |                                       |
|            |  TAB CONTENT (see below)               |

```

- Page padding: 32px (xl)
- Avatar section: flex row, gap 16px (md), avatar 80px rounded-full
- Tab bar: border-b border-[hsl(210,20%,82%)], active tab has gold #C9A84C bottom border 2px + gold text

### Tab: Informations

```
+------------------------------------------+
| Informations de base          [Modifier] |
| ┌──────────────────────────────────────┐ |
| │ Prenom: Felix        Nom: Pitz      │ |
| │ Portable: 06...      Promo: 2024    │ |
| │ Adresse: ...         Ville: ...     │ |
| │ Pole: Developpement                 │ |
| └──────────────────────────────────────┘ |
|                                          |
| Donnees sensibles                        |
| ┌──────────────────────────────────────┐ |
| │ NSS: ****            [Modifier]     │ |
| │ IBAN: ****           [Modifier]     │ |
| └──────────────────────────────────────┘ |
+------------------------------------------+
```

- Two Card sections stacked, gap 24px (lg)
- "Informations de base" card: read mode by default, "Modifier" ghost button top-right toggles to edit mode
- Edit mode: fields become Input components, "Enregistrer" gold CTA + "Annuler" ghost button appear at bottom
- Form layout: 2-column grid on desktop (lg+), single column on mobile
- Field gap: 16px (md) vertical, 16px (md) horizontal in grid
- "Donnees sensibles" card: always read-only display with masked values, each row has its own "Modifier" ghost button that opens a Dialog modal

### Modal: Modifier NSS / IBAN

```
+----------------------------------+
|  Modifier le NSS           [X]  |
|                                  |
|  Nouveau NSS                     |
|  [___________________________]   |
|                                  |
|  Confirmer le NSS                |
|  [___________________________]   |
|                                  |
|  [Annuler]     [Enregistrer]     |
+----------------------------------+
```

- Dialog component (shadcn), max-w-[420px]
- Title: Heading 20px "Modifier le NSS" / "Modifier l'IBAN"
- Two fields: new value + confirmation (must match)
- "Enregistrer" gold CTA, "Annuler" ghost button
- On success: close modal, show Sonner toast "Enregistre avec succes", refresh masked display
- On error: inline error below fields + toast

### Tab: Documents

```
+------------------------------------------+
| Vos documents personnels                 |
|                                          |
| [Ajouter filigrane ↗]  (link, top-right)|
|                                          |
| ┌─────────────────┐  ┌─────────────────┐|
| │ Carte identite  │  │ Carte etudiante │|
| │ [Badge: Uploade]│  │ [Badge:Manquant]│|
| │ [Apercu/lien]   │  │                 │|
| │ [Supprimer]     │  │ [Choisir fichier│|
| └─────────────────┘  └─────────────────┘|
| ┌─────────────────┐  ┌─────────────────┐|
| │ Carte vitale    │  │ Preuve Lydia    │|
| │ ...             │  │ ...             │|
| └─────────────────┘  └─────────────────┘|
| ┌─────────────────┐                      |
| │ RIB             │                      |
| │ ...             │                      |
| └─────────────────┘                      |
+------------------------------------------+
```

- Grid: 2 columns on desktop (lg+), 1 column on mobile
- Gap: 16px (md)
- Each document slot: Card component, min-h-[120px], padding 16px (md)
- Card header: document type name (Body 16px bold 700) + status Badge
- Uploaded state: preview (inline image thumbnail max-h-[160px] or PDF icon + filename), "Telecharger" link (blue), "Supprimer" destructive ghost button (red text)
- Missing state: "Choisir un fichier" button (outline style), accepted formats note at 14px muted
- Upload: standard file input button, max 10 Mo, accepted types: image/*, application/pdf
- "Ajouter filigrane" link: top-right of documents section, opens https://filigrane.beta.gouv.fr in new tab, styled as blue link with external icon

### /dashboard/profil/[userId] (admin view)

- Same layout as own profile
- Page title: "Profil de {prenom} {nom}" instead of "Mon profil"
- Admin can edit basic info (same edit mode)
- Sensitive data section: read-only masked display, NO "Modifier" buttons (admin cannot edit NSS/IBAN of others per 02-CONTEXT.md)
- Documents tab: read-only view (can see uploaded documents, cannot upload/delete)

## Interaction Contracts

### Profile Edit Flow

```
/dashboard/profil → Tab "Informations"
  ├── Read mode (default) — all fields displayed as text
  ├── Click "Modifier" → Edit mode — fields become inputs
  │   ├── "Enregistrer" → validate → PATCH /api/profil → toast success → back to read mode
  │   └── "Annuler" → discard changes → back to read mode
  └── Sensitive fields: click "Modifier" on NSS/IBAN
      └── Opens Dialog modal → double-entry → POST /api/profil/sensitive → toast → close modal
```

### Document Upload Flow

```
/dashboard/profil → Tab "Documents"
  ├── Slot: "Manquant" state
  │   └── Click "Choisir un fichier" → native file picker → validate size/type
  │       ├── Valid → upload to Supabase Storage → show Skeleton during upload → toast success → show preview
  │       └── Invalid → toast error (size or type)
  ├── Slot: "Uploade" state
  │   ├── Preview: inline image (max-h-[160px] object-contain) or PDF icon + filename
  │   ├── "Telecharger" → fetch signed URL → open/download
  │   └── "Supprimer" → confirmation Dialog → DELETE → toast → revert to "Manquant"
  └── "Ajouter un filigrane" → window.open(filigrane URL, '_blank')
```

### Avatar Upload Flow

```
Click avatar or "Changer la photo" link
  → native file picker (image/* only, max 2 Mo)
  → upload to `avatars` bucket → refresh avatar display → toast success
```

### Form Validation

- Validation: onBlur for field-level, onSubmit for full form (consistent with Phase 1)
- Error placement: below each field, 14px DM Sans, text-destructive (#DC2626)
- Toast: Sonner, top-right, 5000ms duration
- Loading states: Skeleton components on initial page load, disabled button + spinner during save/upload

### Responsive Behavior

- Desktop (lg+): 2-column form grid, 2-column document grid
- Mobile (<1024px): single column form, single column document grid
- Modal: max-w-[420px], centered, responsive padding
