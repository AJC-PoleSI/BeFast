## Typography

All sizes are exact — do not substitute.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Display | Playfair Display | 28px | 700 | 1.2 | Page titles (e.g. "Connexion", logo lockup) |
| Heading | Playfair Display | 20px | 600 | 1.2 | Section headings, card titles |
| Body | DM Sans | 16px | 400 | 1.5 | Form labels, paragraph text, sidebar nav items |
| Label / Caption | DM Sans | 14px | 400 | 1.4 | Helper text, error messages, badge text |

Rules:
- Never mix Playfair Display at body size — it is reserved for headings and display only.
- Bold variant of DM Sans (700) is used exclusively for CTA button labels.
- Letter-spacing on Playfair Display headings: -0.01em (tighten slightly for legibility at 20–28px).

## Interaction Contracts

### Form Validation

- Validation triggers: onBlur for field-level, onSubmit for full form
- Error placement: below each field, 14px DM Sans, text-destructive (#DC2626)
- Success: redirect (no visible success toast on login/signup — the redirect IS the confirmation)
- Auth errors from Supabase: displayed via Sonner toast, position top-right, duration 5000ms

### Auth Flow State Machine

```
/login
  ├── submit valid → redirect to /dashboard (or /attente if no role)
  ├── submit invalid → show inline error toast
  └── "Mot de passe oublié ?" → redirect to Supabase reset flow (email sent toast)

/inscription
  ├── submit valid → redirect to /attente (account created, pending role)
  └── submit invalid → field errors + toast for server errors

/attente
  └── "Se déconnecter" → signOut() → redirect to /login

/dashboard
  └── RoleGuard check → if no role → redirect to /attente
                      → if role insufficient → show error message inline
```

### Sidebar Navigation — Role Filtering

- Nav items rendered based on `user.role` permissions from Supabase JWT claims
- Items with no permission are not rendered (not hidden with CSS — completely absent from DOM)
- Active state: compare `usePathname()` to item href, apply gold active styles

### Responsive Behavior

- Desktop (lg+, ≥1024px): sidebar visible, fixed left, content pushes right
- Mobile (<1024px): sidebar hidden by default, hamburger in header reveals it as an overlay drawer
- Auth pages: single-column card, centered — same layout on all screen sizes
